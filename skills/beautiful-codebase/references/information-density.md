# Information Density · 信息密度与读者画像

> **何时读**: Phase 1 Plan 选择 reader profile 时; Phase 4 Step B 写每节时

## 1 · 三个 Reader Profile

### archaeology (考古)
- **读者**: 新接手系统的开发者
- **要什么**: 代码结构全览、关键路径的完整调用链、模块边界清晰
- **密度**: 高 — 每节 2-4 个 prose 段落 + SVG 调用图 + 折叠调用列表 + 优化建议
- **节奏**: 引导式 — 从一个入口点开始,step by step 深入
- **禁忌**: 不要跳步,不要假设读者了解项目术语

### architecture-review (架构评审)
- **读者**: 技术负责人/架构师
- **要什么**: 模块间依赖关系、关键链路的调用拓扑、风险分析
- **密度**: 极高 — 每节 3-5 个 prose + SVG + 折叠列表 + 带有优先级的优化提案
- **节奏**: 扫描式 — 信息块之间逻辑跳跃可接受(读者会扫读)
- **禁忌**: 不要过度解释基本概念;关注关系和边界

### onboarding (上岗引导)
- **读者**: 刚加入项目的新人
- **要什么**: 快速理解项目做什么、核心流程走一遍、在哪改代码
- **密度**: 中 — 每节 1-3 个 prose + 简化流程图 + 明确的"下一步"
- **节奏**: 任务式 — 每节结尾告诉读者"你在本节学到什么,接下来看什么"
- **禁忌**: 不要超 3 层调用展开;不要罗列过多细节

## 2 · 信息密度对照表

| 维度 | archaeology | architecture-review | onboarding |
|------|-----------|-------------------|-----------|
| 每节 prose 段数 | 2-4 | 3-5 | 1-3 |
| SVG 流程图深度 | 完整 | 2 层 | 1 层 |
| 调用列表折叠 | 完整展开 | 仅关键路径 | 仅第 1 层 |
| 优化建议 | 可选 | 必须带优先级 | 简要 |
| SourcePointers | 全部引用文件 | 关键文件 | 入口文件 |
| 建议阅读时间 | 15-30min | 10-20min | 5-10min |

## 3 · Q9b 密度规则(所有 Profile 通用)

- `CodeBlock` ≤ 1 块/节 (Section 04/05 例外 ≤ 2)
- 每块 ≤ 8 行
- 替换优先级: prose → mermaid → table → `<Code inline>` → `<CodeBlock>`
- `<Code inline>` 不计上限,鼓励使用

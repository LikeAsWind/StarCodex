# Reader Profile: Onboarding (上岗引导)

## 读者画像
刚加入项目的新人。不清楚项目整体概念，需要快速上手改第一个 bug / 功能。

## 阅读目标
- "这个项目是做什么的？解决什么业务问题？"
- "核心流程怎么走一遍？入口在哪里？"
- "我想改个简单需求，改哪个文件？"

## 报告节奏
任务式 —— 每节结尾告诉读者"你在本节学到什么，接下来看哪一节"。

## 信息密度
- 每节 1-3 个 prose 段落
- SVG 调用图只展开 1 层
- 调用列表只展示入口层
- 优化建议简要即可
- SourcePointers 只列入口文件

## 约束
- 不展示超过 3 层调用展开
- 不罗列过多细节
- 每节需包含明确的"上手指引"

## 阅读时间
5-10 分钟

## 推荐主题
press 或 terminal。press 暖底亲和，terminal 专业可信。
